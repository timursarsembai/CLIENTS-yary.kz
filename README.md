# ЯРЫ — Лендинг

Лендинг для оптовой продажи древесного угля [yary.kz](https://yary.kz).  
Деплой: [timursarsembai.github.io/CLIENTS-yary.kz](https://timursarsembai.github.io/CLIENTS-yary.kz)

---

## Стек

Vanilla HTML + CSS + JS — один файл `index.html`, без сборщиков и зависимостей.

## Структура

```
index.html        # Весь сайт
assets/           # Изображения
sitemap.xml
robots.txt
CNAME             # yary.kz → GitHub Pages
```

## Функциональность

- **Каталог товаров** — 4 позиции (2, 3, 5, 9 кг), карточки рендерятся JS из массива `PRODUCTS`
- **Корзина** — drawer справа, `localStorage` (`yary_cart`), минимальный заказ 20 мешков
- **Оформление через WhatsApp** — кнопка формирует сообщение с позициями и итогом, корзина очищается после перехода
- **Барабанный выбор количества** — анимированный drum-picker с touch-поддержкой и ленивой подгрузкой чисел
- **Форма обратного звонка** — попап, отправляет номер в WhatsApp
- **Двуязычность (RU / ҚАЗ)** — переключатель в хедере и мобильном меню, язык сохраняется в `localStorage` (`yary_lang`)
- **Scroll-reveal анимации** — Intersection Observer, плавное появление секций
- **Lazy loading** изображений ниже fold
- **Sticky мобильный CTA** — появляется после прокрутки мимо каталога
- **Кнопка "Наверх"** — появляется после 400px скролла
- **FAQ-аккордеон** — 6 вопросов, двуязычный массив `FAQS`
- **SEO** — meta description, Open Graph, Twitter Card, Schema.org (LocalBusiness, FAQPage, ItemList)

## Локализация

Все строки интерфейса в словаре `I18N` (объект с ключами `ru` и `kk`).  
Статический контент переключается через `data-i18n` / `data-i18n-html` атрибуты.  
Динамический контент (карточки товаров, FAQ, корзина) перерендеривается при смене языка.

## Разработка

Открыть `index.html` напрямую в браузере — внешних зависимостей нет.  
Для live-reload можно использовать любой локальный сервер, например:

```bash
npx serve .
# или
python -m http.server 3456
```

## Деплой

GitHub Pages из ветки `main`. Домен привязан через `CNAME`.

```bash
git add .
git commit -m "описание изменений"
git pull --rebase
git push
```

Изменения появляются на сайте через 1–2 минуты.
